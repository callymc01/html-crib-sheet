Forms

This page summarises the form elements and attributes I've learned so far.

Action & Method Attributes

The <form> element usually begins with two important attributes:

  • action — where the form data is sent when submitted
  • method — the HTTP method used (commonly GET or POST)
I haven’t learned HTTP methods yet, but I know GET and POST are the most common.

Fieldset & Legend

<fieldset> groups related form controls together and draws a box around them.

<legend> provides a title for that group.

Show example 

<fieldset>
    <legend>Personal Info</legend>
    <label for="name">Name:</label>
    <input id="name" type="text">
</fieldset>

Label Element & For Attribute

The <label> element connects text to an input.

The for attribute must match the input’s id exactly.

Show example 

<label for="input-id">Label text</label>
<input id="input-id" type="text">

Input Elements

The <input> element is a void element (no closing tag) and changes behaviour based on its type.

Common Types

  • text — single-line text field
  • email — checks email format
  • number — numeric input only
  • radio — one option from a group (same name)
  • checkbox — multiple independent selections

Useful Input Attributes

  • id — unique identifier, links to label
  • name — key used when sending form data
  • placeholder — hint text
  • min / max — restricts allowed range
  • value — default value or submitted value

Select Element (Dropdown)

<select> creates a dropdown list. Inside it are <option> elements.

  • select is the container
  • option defines each choice
  • value is what gets submitted
  • the text between <option> tags is what the user sees
  • selected sets a default option

Button Element

Buttons perform actions. The type attribute defines what they do:

  • submit — submits the form
  • reset — resets all fields
